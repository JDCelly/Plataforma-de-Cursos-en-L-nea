Resumen: El Flujo Completo de Archivos

Paso Archivo ¿Qué Hace? Comunicación
1app/…/page.tsxEl usuario interactúa con el formulario y llama handleSubmit()Llamada a función
2services/….service.tsPrepara los datos y llama api.post() / api.get()Llamada a función
3lib/api.tsConstruye URL, ejecuta fetch() por la redHTTP (red)
4….controller.tsRecibe la petición HTTP, extrae @Body/@ParamInyección DI
5….service.ts (BE)Valida negocio, llama prisma.modelo.create/find/…Prisma Client
6prisma.service.tsTraduce query Prisma a SQL y envía a PostgreSQLSQL (TCP)
7PostgreSQLEjecuta el SQL y retorna registrosResultado SQL

Punto clave

Frontend y Backend son aplicaciones separadas que solo se comunican por HTTP. Dentro de cada aplicación, los archivos se comunican por importaciones y llamadas a funciones. El backend habla con PostgreSQL a través de Prisma (SQL sobre TCP).
